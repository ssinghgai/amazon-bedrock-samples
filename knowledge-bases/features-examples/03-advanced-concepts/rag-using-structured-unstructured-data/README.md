# Multiple Retrievers in a QA Chain

## Context

The MultiRetrievalQAChain is an advanced concept inspired by the Retrieval-Augmented Generation (RAG) approach, which combines the strengths of retrieval-based and generation-based language models. It allows for the integration of multiple retrievers, each specialized in a different data source, enabling the system to leverage diverse information sources to generate comprehensive and accurate responses.

## Description

The MultiRetrievalQAChain combines retrieval-based and generation-based language models, allowing the integration of multiple retrievers, each specialized in a different data source. This enables the system to leverage diverse information sources to generate comprehensive and accurate responses.

The notebook sets up a MultiRetrievalQAChain that can answer queries by retrieving relevant information from a knowledge base and a database, keeping track of conversation context, and synthesizing multiple sources to provide natural language answers.

## Important Notes

1. **Custom Module for Amazon Athena**: The notebook uses a custom module built specifically for Amazon Athena, but you can customize it for other databases like Amazon Redshift and Amazon RDS using data APIs.

2. **Knowledge Base and Data Availability**: You need to have a Knowledge Base for Amazon Bedrock and a SQL database available. If you don't have a SQL database, you can use the optional notebook `create-sql-dataset (Optional).ipynb` to set up Amazon Athena with dummy data.

## Setting up the Custom Retriever for Text-to-SQL

The code for the custom module can be found in `CustomDatabaseRetriever.py`. This module defines a retriever class called `AmazonAthenaRetriever` that retrieves relevant data from an Amazon Athena database using SQL queries generated by Amazon Bedrock.

The retriever interacts with the Athena database through the AWS boto3 SDK, allowing running SQL queries on data stored in Amazon S3. It generates SQL queries based on natural language input, executes the queries on Athena, and returns the results as a list of documents formatted for a LangChain RetrievalQA chain.
